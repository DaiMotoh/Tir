  • Natural Coroutines Dead easy coroutine style state management with a natural flow.
  • Handler Style Fast "handler" style objects for building your own custom state.
  • Callback Style Like callbacks or think that's the best model for the job? Tir's got that too.
  • Easy To Use Even though it supports these different styles they're all easy to use.
  • Tiny Code A complete framework in 1300 lines of Lua thanks to Mongrel2.
  • Easy Testing Easily write automated tests for your site, without a lot of boilerplate.
  • Already Scalable Tir uses a process per page model which means it's already scaled out in one direction.
  • Simple Structure The most recent hybi WebSocket protocol.
  • Mongrel2 Based Mongrel2 handles all the crap at the edge like routing and static files.
  • ZeroMQ Based Talk to any language that can talk ZeroMQ.
  • Background Tasks Async tasks already built into the framework for off-loading background tasks.
  • Pragmatic Polyglot Tir's design lets you easily replace pieces of your web app with any other language. No more polishing turds.
